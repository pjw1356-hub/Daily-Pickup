# 📊 데일리 픽업 (Daily Pickup) 텔레그램 알림 시스템 지식 베이스

이 문서는 매일 아침 주식 추천 종목을 텔레그램으로 자동 전송하는 시스템의 구축 과정과 주요 해결 사항을 정리한 문서입니다.

## 📌 프로젝트 개요
- **목적**: 매일 아침 주식 개장 전(아침 8:00)에 오늘의 추천 종목을 텔레그램 메시지로 자동 수신.
- **주요 기술**: Python, JavaScript, GitHub Actions, Telegram Bot API.

## 📁 주요 구성 파일
- **`app.js`**: 주식 데이터(KOSPI, KOSDAQ)가 담겨 있는 소스 파일.
- **`send_stocks.py`**: `app.js`에서 데이터를 정규식으로 읽어와 텔레그램으로 전송하는 핵심 파이썬 스크립트.
- **`send_stocks.yml`**: GitHub Actions에서 매일 아침 8시(월~금)에 스크립트를 실행하도록 설정한 워크플로우 파일.
- **`push.bat`**: 윈도우 환경에서 수정된 코드를 깃허브에 쉽게 올리기 위한 배치 파일.

## 💡 주요 문제 해결 (Gotchas)

> [!NOTE]
> **1. 메시지 포맷 문제 (HTML)**
> 텔레그램 메시지에서 글씨를 두껍게(`<b>`) 표시하기 위해 `parse_mode`를 `"HTML"`로 설정했습니다. 이로 인해 가독성이 대폭 향상되었습니다.

> [!IMPORTANT]
> **2. 명령어 환경 차이 (Windows)**
> 사용자님의 컴퓨터 환경에서는 `python` 명령어 대신 `py` 명령어를 사용해야 파이썬 스크립트가 정상 작동합니다. (예: `py send_stocks.py`)
> *참고: 깃허브 서버는 리눅스 기반이므로 그대로 `python`을 사용합니다.*

> [!WARNING]
> **3. GitHub Secrets 빈 값 문제**
> 깃허브의 Secrets가 비어있을 때 Python의 `os.environ.get()`이 `None`이 아닌 빈 문자열(`""`)을 반환하여 기본값이 무시되는 문제가 있었습니다. 이를 해결하기 위해 `or` 연산자를 사용하여 안전하게 기본값이 적용되도록 수정했습니다.

> [!CAUTION]
> **4. 최종 해결: 보안을 위한 GitHub Secrets 사용**
> 보안 토큰이 코드에 포함되면 깃허브 푸시가 차단(Push Protection)됩니다. 따라서 하드코딩을 제거하고 다시 **GitHub Secrets**를 사용하도록 변경했습니다. 만약 작동하지 않는다면 깃허브 저장소 설정(Settings > Secrets and variables > Actions)에서 토큰 값이 정확한지 꼭 확인해야 합니다.

## 🚀 앞으로의 사용 방법
1. **코드 수정 시**: 로컬에서 `app.js`나 `send_stocks.py`를 수정한 후, 터미널에서 `.\push.bat`을 실행하여 깃허브에 반영합니다.
2. **자동 실행**: 매일 아침 8시(월~금)에 자동으로 텔레그램 메시지가 발송됩니다.
